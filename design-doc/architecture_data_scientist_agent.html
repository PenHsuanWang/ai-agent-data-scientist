🔄 ReAct Agent Loop (DataScienceAgentService)

Text-based Thought → Action → Observation. NOT native tool-use API.
1. _build_system_prompt() — assembles tool summary for all 16 tools + 12 numbered workflow & code-generation rules.
2. _client.messages.create(model, max_tokens, system=prompt, messages=session.messages) — sends full message history. No tools= parameter.
3. _parse_react(raw_text) — regex extracts Thought / Action / Action Input or Final Answer. Returns {"type": "action"|"final_answer"|"parse_error", ...}.
4. On "parse_error": if count < 3 → inject correction prompt and retry; if count ≥ 3 → raise ReActLoopError.
5. On "action": tool_registry[action_name](action_input) → observation: str. All tools return str, never raise.
6. session.add_assistant_message(thought+action) + session.add_user_message("Observation: ..."). Loop back to step 2.
7. On "final_answer": exit loop, return answer. Hard limit: MAX_REACT_ITERATIONS = 20. Raises ReActLoopError if exceeded.

Flow (Order of Operations)

1 User sends POST /api/v1/analysis/chat with {message, session_id}.
2 AnalysisSessionStore.get_or_create(session_id) returns existing or new AnalysisSession. DataScienceAgentService.run(session, message) is called.
3 ReAct loop: _build_system_prompt() (16-tool summary + 12 rules) → _client.messages.create() with full session message history. No tools=.
4 _parse_react(raw_text) extracts Thought / Action / Action Input. tool_registry[action_name](action_input) dispatches to one of 16 tools. Observation appended; loop continues (max 20 iterations).
5 Physical validation tools (validate_physical_units, check_magnitude) are called by Claude as needed; results logged to session.unit_context.
6 On Final Answer, loop exits. Returns AnalysisResponse (response, session_id, react_trace, figures, unit_validations, iterations_used, model, status).
7 Client retrieves artifacts: GET /api/v1/analysis/{session_id}/figures/{figure_id} → PNG; GET /api/v1/analysis/{session_id}/notebook → .ipynb.
Design Highlights
• Text-based ReAct: Thought → Action → Observation — NOT native tool-use API (tools= never passed to Claude).
• Tools never raise: all 16 tools return str; errors returned as "Error: ..." strings; Claude self-corrects.
• Path traversal protection: all file access via _safe_resolve(base_dir, file_name).
• Physical law enforcement: pint DOMAIN_RANGES (15 quantities) rejects e.g. efficiency > 100%.
• plt.show() patched: figures captured as base64 PNG via __FIGURES__ sentinel in subprocess stdout.
• Full audit trail: react_trace (Thought/Action/Observation) returned in every AnalysisResponse.
• Pluggable backends: SubprocessCodeRunner (default, stateless) or JupyterKernelManager (persistent) — set via CODE_EXECUTION_BACKEND.
• Notebook export: any session exportable as .ipynb via export_notebook(title) tool.
